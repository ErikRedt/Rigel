Categoría Tipo Patrón de diseño Caracteristicas Principales
Creacionales De clase Abstract Factory Crea diferentes familias de objetos.
Builder Construye objetos complejos de forma incremental.
De objeto Factory Method Permite que las subclases determinen clases a instanciar.
Prototype Clonar nuevos ejemplares de un prototipo.
Singleton Restringe la instanciación de un tipo a un solo objeto.
Objet Pool Asegura que un solo hay un miembro de una clase y da acceso global a el.
Model View Controller divide la creacion en modelo-vista-controlador permitiendo la creación en las 3.
Estructurales De clase Class adapter Covierte una interfaz en otra.
Herencia múltiple Una clase hereda de otras y combina propiedades.
De objetos Object adapter Estructurales
Bridge Desacopla la abstracción de la implementación permitiendo modificarlas independientemente.
Composite Construye objetos complejos a partir de simples, de forma recursiva y con estructura de arbol.
Decorator Permite añadir dinamicamente fucionalidades a clases existentes.
Facade Permite simplificar la interfaz de un sistema.
Flyweight Elimina redundancia o la reduce.
Proxy Un objeto que se aproxima a otro.
De comportamiento De clase Interpreter Interprete de lenguaje para una gramatica simple y sencilla.
Template method Algoritmo con varios pasos suministrados por una clase derivada.
De objeto Chain of responsability La base es permitir que más de un objeto tenga la posibilidad de atender una petición.
Command Encapsula una petición como un objeto dando la posibilidad de “deshacer” la petición.
Iterator Define una interfaz que declara los métodos necesarios para acceder secuencialmente a una colección de objetos sin exponer su estructura interna.
Mediator Coordina las relaciones entre sus asociados. Permite la interacción de varios objetos, sin generar acoples fuertes en esas relaciones.
Memento Almacena el estado de un objeto y lo restaura posteriormente.
Observer Notificaciones de cambios de estado de un objeto.
Server Se utiliza cuando el comportamiento de un objeto cambia dependiendo del estado del mismo.
State Objeto cuyo comportamiento depende de un estado (el objeto puede aparentar cambiar de clase según un estado).
Strategy Utilizado para manejar la selección de un algoritmo.
Visitor Operaciones aplicadas a elementos de una estructura de objetos heterogénea.
